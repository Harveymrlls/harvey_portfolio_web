HarvsGallery Complete Workflow Summary

1. Authentication & Entry

  • User accesses website → index.html loads
  • User chooses between:
    • Login: Existing credentials verification
    • Signup: New account creation with username availability check
  • Successful authentication creates PHP session with user data
  • User redirected to main gallery interface

2. Gallery Initialization

  • gallery.php loads and validates user session
  • JavaScript fetches user's media data via gallery_fetch.php
  • Database query retrieves user's uploads from uploads table
  • Dynamic rendering creates gallery grid with images and videos
  • Gallery ready for user interactions

3. File Upload Process

  • User selects file through upload interface
  • Client-side validation: File type and size checking
  • Progress bar displays during upload
  • Server-side processing (upload.php):
    • Session validation
    • File type verification
    • Move file to uploads/ directory
    • Store metadata in database
  • Success/error notification displayed
  • Gallery automatically refreshes

4. Media Viewing

  • User clicks on media item
  • System detects file type (image/video)
  • Opens appropriate modal viewer:
    • Images: Full-size display
    • Videos: Embedded player with controls
  • User closes modal to return to gallery

5. File Download

  • User clicks download button on media item
  • System verifies file ownership
  • Browser initiates file download
  • Original filename preserved

6. File Deletion

  • User clicks delete button
  • Confirmation dialog appears
  • Server processing (delete.php):
    • Ownership verification
    • Physical file deletion from server
    • Database record removal
  • Success notification shown
  • Gallery refreshes to reflect changes

7. Session Management

  • Every page request validates session
  • Automatic redirect to login if session invalid
  • Session timeout after period of inactivity
  • Manual logout destroys session and redirects to login

8. Data Flow

  • User Data: Registration → Database (users table) → Session storage
  • File Upload: Client → Server validation → File system → Database → UI update
  • Media Retrieval: Database query → JSON response → JavaScript rendering
  • Security: Session checks → Ownership verification → File validation

9. Security Layers

  • Password hashing for user accounts
  • Session-based authentication
  • File ownership verification before operations
  • Client and server-side file validation
  • SQL injection prevention via prepared statements
  • Cross-user data isolation
